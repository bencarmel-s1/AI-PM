# Template: Competitive Analysis

## Before you start

**What you need:** The names of competitors you want to analyze, and a question you're trying to answer (e.g. "Should we build X or is it already table stakes?").

**Optional input:** Any research you've already done — save competitor notes, pricing pages, or feature docs as `.txt` files and reference them.

**What Claude can do without files:** Claude has broad knowledge of well-known SaaS products and can fill in a competitive matrix from its training data. For niche or very recent competitors, provide your own research files.

---

## How to use
`Research competitors for [feature/product] using @competitive-analysis.md`

Or with your own research: `Analyze @competitor-notes.txt and structure it using @competitive-analysis.md`
