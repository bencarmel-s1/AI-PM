# Pulse — Product Reference

## What We Build

**Pulse** is a user behavior analytics and monitoring platform for B2B SaaS teams. Product teams use Pulse to understand how users interact with their product, diagnose friction, and measure the impact of changes.

**Primary users:** Product managers, UX researchers, growth engineers, data analysts
**Deployment model:** SaaS (cloud-hosted), with optional self-hosted SDK for event capture
**Pricing:** Seat-based + usage tiers (events/month)

---

## Product Pillars

| Pillar | Description | Key Features |
|--------|-------------|-------------|
| **Analytics** | Understand user behavior with event-based data | Funnel analysis, retention curves, cohort analysis, custom dashboards |
| **Session Replay** | Watch how real users interact with your product | Full session playback, rage-click detection, error overlays |
| **Alerts** | Proactive monitoring of product health | Custom metric alerts, anomaly detection, Slack/email digest |
| **Integrations** | Connect Pulse to your existing stack | 40+ native integrations, webhook support, custom event API |

---

## Core Differentiators

1. **Ease of setup** — median time to first event is 4 hours with our JS snippet
2. **Qualitative + quantitative in one place** — session replay linked to funnel drop-offs
3. **Team collaboration** — shareable dashboards, annotation layer, Slack digest
4. **Non-technical friendly** — no SQL required; drag-and-drop funnel builder

---

## Key Metrics We Track

| Metric | Definition |
|--------|-----------|
| MAU | Monthly active users on the Pulse platform |
| Time-to-first-value | Days from signup to first meaningful insight action |
| Feature adoption | % of MAU using each pillar feature ≥ 1x per month |
| D30 retention | % of users still active 30 days after signup |
| Integrations per account | Average active integrations per paying customer |
