## Quick Start (30 seconds)

Open Claude Code in your project and paste this:

```
I need to develop a product strategy for [feature/product area].
Use @product-strategy-workflow.md and @rumelt-strategy-kernel.md to guide me.
My context: [2-3 sentences about your product, team size, constraints]
The strategic question I'm answering: [What leadership asked you]
```

Claude will guide you through competitive research, strategic choices with devil's advocate pressure-testing, and synthesize everything into a complete strategy document and executive presentation.

## The 3-Phase Process

```
Phase 1 — Diagnose     → Research competitive landscape, identify the real challenge
Phase 2 — Make Choices → 5 strategic decisions with devil's advocate pressure-testing
Phase 3 — Synthesize   → Complete strategy doc + executive slide deck
```

Full details in `product-strategy-workflow.md`.

## The Framework: Rumelt's Strategy Kernel

```
DIAGNOSIS      → What's really going on? (Competitive landscape + constraints)
GUIDING POLICY → What's our approach? (Hard choices about where to compete and how to win)
COHERENT ACTIONS → How do we execute? (Roadmap where every piece reinforces the others)
```

The most common mistake: jumping from diagnosis straight to a feature list, skipping the guiding policy. Real strategy involves saying NO to things.

**The test:** Would a reasonable person disagree with your guiding policy? If not, it's probably too generic.

## Which Framework to Use?

| Situation | Framework |
|-----------|-----------|
| Developing strategy from scratch | Rumelt's Strategy Kernel (`rumelt-strategy-kernel.md`) |
| Auditing an existing strategy | SWOT Analysis (`swot-analysis.md`) |
| Pressure-testing strategic choices | Gibson-Biddle DHM (`gibson-biddle-dhm.md`) |
| All three together | Start with SWOT for situation, Rumelt for strategy, DHM to pressure-test |
