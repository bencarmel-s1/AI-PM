# Execution Prompts

Prompts for PRD writing, spec generation, and ticket creation.

Use these when you're translating a strategy or decision into something the team can actually build.

---

## What Belongs Here

- PRD drafting and refinement
- User story and acceptance criteria writing
- Engineering ticket creation (Jira, Linear, etc.)
- Edge case and error state identification
- Spec review and gap analysis
- Release notes and changelog drafting

---

## How to Add a Prompt

1. Create a new `.md` file in this folder (e.g., `prd-draft.md`)
2. Follow the prompt writing guidelines in the [main README](../../README.md#-prompt-writing-guidelines)
3. Include: purpose, when to use it, the prompt itself, and a usage example

---

## Prompts in This Folder

| Prompt | What it does |
|---|---|
| [`prd-draft.md`](./prd-draft.md) | Draft a PRD section from a feature brief or research synthesis |

---

> **Tip:** Execution prompts get much better output when you include your team's existing format or template as context. Paste in an example PRD or ticket and ask Claude to match the style.
